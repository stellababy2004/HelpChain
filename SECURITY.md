# Security Policy

## Security Overview

HelpChain applies a defence-in-depth approach combining controlled access, operational traceability, infrastructure hardening and secure deployment practices.

Security, confidentiality and governance are treated as core platform requirements throughout the product lifecycle.

---

## Responsible Disclosure

If you believe you have identified a security vulnerability or sensitive issue related to HelpChain, please contact us privately.

### Contact

- Security contact: security@helpchain.live
- Preferred languages: French or English

Please include:

- a clear description of the issue;
- reproduction steps if applicable;
- potential impact assessment;
- relevant logs or screenshots if available.

Please do not disclose vulnerabilities publicly before coordinated remediation.

---

## Security Principles

HelpChain is designed around the following principles:

- least-privilege access control;
- role-scoped visibility;
- operational traceability;
- secure-by-default configurations;
- progressive hardening of deployments;
- privacy-aware data handling.

---

## Platform Security Measures

### Application Security

- HTTPS enforcement and secure transport policies;
- Content Security Policy (CSP);
- CSRF protection on sensitive operations;
- server-side validation and sanitisation;
- rate limiting on sensitive endpoints;
- secure session handling.

### Authentication and Access Control

- protected administrative access;
- secure password handling;
- session protection and renewal;
- multi-factor authentication support for privileged accounts;
- scoped permissions and operational roles.

### Data Protection

- controlled access to operational data;
- audit-oriented activity logging;
- minimisation of exposed personal data;
- secure handling of sensitive information.

### Infrastructure and Monitoring

- dependency and vulnerability scanning;
- automated security analysis in CI pipelines;
- monitoring and operational alerting;
- controlled deployment workflows.

---

## Security Updates

Critical security issues are prioritised and handled according to operational severity and impact.

Security-related dependencies and infrastructure components are reviewed regularly as part of the deployment workflow.

---

## Compliance and Governance

HelpChain is developed with privacy, traceability and governance considerations integrated into the platform architecture.

This includes:

- GDPR-oriented operational practices;
- audit-oriented logging;
- controlled access management;
- security-focused deployment policies.
