# Двустепенна автентикация (2FA) за HelpChain

## Обзор
Имплементирана е пълна система за двустепенна автентикация (2FA) за административни акаунти в HelpChain приложението. Системата използва TOTP (Time-based One-Time Password) стандарта и е съвместима с популярни приложения като Google Authenticator, Authy, Microsoft Authenticator и други.

## Основни функционалности

### 1. TOTP (Time-based One-Time Password)
- **Алгоритъм**: SHA1
- **Дължина на код**: 6 цифри
- **Валидност**: 30 секунди
- **Толеранс**: ±30 секунди (1 интервал назад/напред)

### 2. QR код генериране
- Автоматично генериране на QR кодове за лесна настройка
- Съвместимост с всички основни 2FA приложения
- Base64 енкодиране за показване в браузъра

### 3. Backup кодове
- **10 кода за еднократна употреба**
- Формат: XXXX-XXXX (4-4 цифри)
- Криптографски сигурни (използва `secrets` модула)
- Възможност за регенериране

### 4. Интеграция с логин процеса
- Плавна интеграция с съществуващия админ логин
- Двустепенен процес: парола → 2FA код
- Автоматично препращане при липсващ 2FA код

## Административен интерфейс

### Настройка на 2FA (`/admin_2fa_setup`)

#### Стъпка 1: Активиране
- Бутон "Активирай 2FA" за започване на процеса
- Автоматично генериране на TOTP secret ключ

#### Стъпка 2: Сканиране и потвърждение  
- **QR код** за сканиране с мобилно приложение
- **Ръчно въвеждане** на secret ключа (backup опция)
- **Потвърждение** с първия TOTP код
- Показване на backup кодовете

#### Стъпка 3: Завършване
- Потвърждение за успешна активация
- Напомняне за запазване на backup кодовете
- Опция за отпечатване на кодовете

### Управление на backup кодове (`/admin_2fa_backup_codes`)
- **Преглед** на всички backup кодове
- **Копиране** на кодове с едно кликване
- **Изтегляне** като текстов файл
- **Отпечатване** на кодовете
- **Регенериране** с 2FA потвърждение

### Деактивиране на 2FA
- Безопасно деактивиране с TOTP потвърждение
- Изчистване на всички 2FA данни
- Предупреждения за намалена сигурност

### Основни функции

#### `generate_totp_secret()`
```python
def generate_totp_secret():
    """Генерира TOTP secret ключ"""
    return pyotp.random_base32()
```

#### `verify_totp_token(admin_user, token)`
```python
def verify_totp_token(admin_user, token):
    """Проверява TOTP токен"""
    totp = pyotp.TOTP(admin_user.totp_secret)
    return totp.verify(token, valid_window=1)
```

#### `generate_backup_codes(count=10)`
```python
def generate_backup_codes(count=10):
    """Генерира backup кодове за 2FA"""
    codes = []
    for _ in range(count):
        code = ''.join(secrets.choice('0123456789') for _ in range(8))
        codes.append(f"{code[:4]}-{code[4:]}")
    return codes
```

## Сигурност

### Защита на данните
- **Secret ключове** се съхраняват сигурно в базата данни
- **Backup кодове** са хеширани JSON в базата данни
- **TOTP токени** имат ограничена валидност (30 сек)

### Защита срещу атаки
- **Rate limiting** чрез неуспешни опити за вход
- **Заключване на акаунти** при повторни неуспешни опити  
- **Криптографски сигурни** backup кодове
- **IP и User-Agent** логване за одитиране

### Backup опции
- **10 backup кода** за всеки потребител
- **Еднократна употреба** на всеки backup код
- **Възможност за регенериране** с 2FA потвърждение

## Workflow за потребители

### Първоначална настройка
1. **Вход в админ панела** с потребителско име и парола
2. **Отиване до "2FA настройки"** от главното меню
3. **Кликване на "Активирай 2FA"**
4. **Сканиране на QR кода** с мобилно приложение
5. **Въвеждане на първия 6-цифрен код** за потвърждение
6. **Запазване на backup кодовете** на сигурно място

### Ежедневна употреба
1. **Въвеждане на потребителско име и парола**
2. **Въвеждане на 6-цифрения код** от мобилното приложение
3. **Автоматично влизане** в админ панела

### Възстановяване при загуба
1. **Използване на backup код** вместо TOTP код
2. **Регенериране на нови backup кодове** след влизане
3. **Деактивиране и повторна настройка** на 2FA при нужда
