## INCIDENT REPORT — 2025-12-12

Summary
-------
- Incident: potential secret leakage from Vercel prebuild artifacts discovered in CI.
- Repository: stellababy2004/HelpChain.bg
- Reported / detected: 2025-12-12
- Owner / responder: repository maintainer (solo)

Scope & Impact
--------------
- Prebuilt artifacts produced by `npx vercel build` contained Python package metadata (`*.dist-info`) and other vendored files. Scanner flagged potential secrets inside artifact contents.
- Action taken prevented further public exposure (artifacts deleted where necessary, tokens rotated).

Actions Performed
-----------------
1. Immediate containment
   - Rotated / revoked exposed JWTs and any other affected credentials.
   - Deleted public artifacts that contained sensitive data.

2. Remediation and CI hardening
   - Added `.gitleaks.toml` allowlist tuned for vendor/site-packages and `.dist-info` (converted to re2 regex to avoid gitleaks panics).
   - Updated `.github/workflows/prebuild-and-deploy.yml` to create a sanitized copy `.vercel/output-scrubbed` and run gitleaks against that copy; preserved the original `.vercel/output` for deploy.
   - Added logic to ensure placeholder `INSTALLER` files exist under `.dist-info` in the prebuilt venv so Vercel deploy doesn't fail.
   - Pushed redaction commits to remove test secrets from repo where found.

3. Verification
   - Triggered CI run on branch `chore/remove-distinfo-and-add-gitleaks` (run id: 20165870895).
   - Downloaded `gitleaks-results.sarif` from the run artifacts and ran local parser.
   - Result: `Total findings: 0` (gitleaks reported no leaks in final SARIF).

Files changed (high level)
-------------------------
- `.github/workflows/prebuild-and-deploy.yml` — added sanitized-scan flow, venv handling.
- `.gitleaks.toml` — converted patterns to re2-compatible allowlist.
