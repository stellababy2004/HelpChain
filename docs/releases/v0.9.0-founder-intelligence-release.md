# v0.9.0 Founder Intelligence Release

Internal release document for the HelpChain founder/admin operating layer.

## Release Scope

This release connects institutional onboarding, premium professional lead handling, first-party audience telemetry, Revenue Radar, and anonymous-to-lead attribution.

It expands HelpChain from an operational request-management platform into a founder intelligence loop:

- Capture institutional access demand.
- Review and convert approved organizations into structures.
- Restore professional lead pipeline visibility.
- Track high-intent public page activity.
- Rank anonymous sessions by likely business intent.
- Attach pre-conversion audience context to later access requests and professional leads.

## Institutional Onboarding Phase 1

### Added

- Public institutional access request page:

```text
/demander-acces
```

- `OrganizationAccessRequest` model backed by the `organization_access_requests` table.
- Admin list page for access requests:

```text
/admin/organizations/requests
```

- Admin navigation/discoverability entry for institutional access requests.

### Product Impact

HelpChain now has a formal institutional intake path. Founder/admin users can capture organization name, contact details, city, organization type, estimated users, message, review status, and internal notes in one workflow instead of handling access demand outside the product.

## Institutional Onboarding Phase 2

### Added

- Access request detail page:

```text
/admin/organizations/requests/<id>
```

- Review actions:

```text
/admin/organizations/requests/<id>/approve
/admin/organizations/requests/<id>/reject
/admin/organizations/requests/<id>/need-info
```

- Structure creation on approval.
- First organization admin creation on approval.
- Internal notes handling during review.
- Review metadata for reviewer and reviewed timestamp.
- Duplicate approval protection so re-approving an approved request does not create another structure.

### Product Impact

Institutional access requests can now move from intake to operational onboarding. Approved requests become tenant structures with an initial admin user, while rejected and need-info requests remain traceable.

## Premium Professional Leads Restore

### Added

- Restored professional leads admin route:

```text
/admin/professional-leads
```

- Premium pipeline view for professional lead qualification.
- KPI strip for new leads, leads to qualify, priority leads, follow-ups due, and qualified leads.
- Improved filtering by email, profession, city, and status.
- Improved list/table experience for lead state, urgency, source, last activity, and next action.
- Responsive card presentation for smaller screens.

### Product Impact

Professional leads are again visible as a structured founder/admin pipeline. This gives the founder a practical surface for reviewing partner, professional, and referral opportunities instead of leaving them as scattered form submissions or imports.

## Audience France Map

### Added

- Admin audience map:

```text
/admin/audience-map
```

- KPI strip for recent visitors, active cities, key page views, and detected intent signals.
- France-first territorial audience view with city markers when location data is available.
- Top pages table based on recent `page_view` traffic.
- Founder intelligence section covering business-readable signals, sources, and repeated sessions.
- Graceful no-data handling when analytics tables or usable geo data are unavailable.

### Product Impact

The founder now has a practical internal view of territorial and behavioral interest in HelpChain. The page helps identify where interest is appearing, which public pages are being viewed, and whether high-intent behavior is repeating.

## Audience Data Feed

### Added

- Server-side `page_view` ingestion for selected public pages.
- Tracked public pages:

```text
/
/offre
/deploiement
/professionnels
/demander-acces
/contact
```

- Reuse of existing analytics models:

```text
AnalyticsEvent
UserBehavior
```

- Analytics schema migration for:

```text
analytics_events
user_behaviors
```

- Defensive tracking behavior so analytics write failures do not break page rendering.

### Product Impact

HelpChain can collect lightweight first-party telemetry for high-intent pages before a visitor logs in or submits a form. This data powers the Audience Map, Revenue Radar, and pre-conversion prospect context.

## Revenue Radar

### Added

- Anonymous session scoring based on:
  - pages visited
  - page count
  - recency
  - repeat behavior
  - external referrer signals

- Temperature buckets:
  - Froid
  - Tiede
  - Chaud
  - Tres chaud

- Recommended founder actions in the Audience Map.
- Founder insights summarizing high-intent traffic.
- Session ranking by likely business intent.

### Product Impact

Revenue Radar gives the founder a prioritized view of anonymous sessions that appear commercially relevant. It does not identify visitors by itself; it ranks behavior so founder attention can focus on high-intent sessions.

## Prospect CRM Auto Capture

### Added

- Linking anonymous session history to later form submissions when the same session converts.
- Target models:

```text
OrganizationAccessRequest
ProfessionalLead
```

- "Audience avant conversion" intelligence on admin detail pages.
- Stored conversion context including:
  - session score
  - temperature
  - source
  - first seen timestamp
  - last seen timestamp
  - pages viewed
  - key pages
  - repeat visit signal
  - intent flags

- Revenue Radar linkage badges for sessions already tied to an access request or professional lead.

### Product Impact

Founder/admin review can now include what a prospect did before converting. For example, an institutional access request can show prior visits to `/offre`, `/deploiement`, or `/demander-acces`, making qualification more informed without manually reconstructing raw analytics rows.

## Product Impact Summary

This release connects previously separate founder workflows into one early institutional growth loop:

- Institutions can request access publicly.
- Admins can review and convert approved institutions into tenant structures.
- Professional leads can be managed in a restored premium pipeline.
- First-party audience signals from key public pages inform founder prioritization.
- Anonymous high-intent sessions can later be attached to access requests or professional leads.

The result is a clearer path from anonymous interest to captured prospect to reviewed institutional onboarding.

## Deferred / Not Yet Implemented

- Automated outbound email workflow for access request approval, rejection, or need-info decisions.
- Automated password delivery or activation flow for the first organization admin.
- Full CRM ownership model beyond currently visible lead/admin fields.
- External analytics provider integration.
- Real-world organization identification for anonymous sessions before form submission.
- Additional schema changes beyond the implemented access request and analytics migrations.

## Recommended Next Priorities

- Add a safe activation or invitation flow for the first organization admin created during approval.
- Add explicit operator/founder playbook steps for reviewing access requests and acting on Revenue Radar sessions.
- Add lightweight status history or audit visibility for access request review decisions if deeper traceability is needed.
- Define whether professional leads and organization access requests should converge or remain separate in the product model.
- Add a retention/privacy note for audience session intelligence attached to prospect records.
